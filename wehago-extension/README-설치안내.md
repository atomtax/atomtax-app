# 아톰베이스 위하고 수집기 (크롬 확장)

위하고T Smart A 화면을 열면, 그 **응답 데이터만 읽어** 아톰베이스로 자동 동기화하는 확장입니다.
위하고로는 어떤 요청도 보내지 않으며(읽기 전용), 화면 동작도 바꾸지 않습니다.

> 상세 설치 가이드(스크린샷 포함)는 2단계-C에서 별도 제공됩니다. 아래는 최소 안내입니다.

## 설치 (개발자 모드 — 웹스토어 아님)

1. 크롬 주소창에 `chrome://extensions` 입력
2. 우측 상단 **개발자 모드** 켜기
3. **압축해제된 확장 프로그램을 로드합니다** 클릭 → 이 `wehago-extension/` 폴더 선택
4. 확장 아이콘(보라 원자)을 클릭 → 팝업에서:
   - **직원 토큰** 붙여넣기 후 [저장] (토큰은 아톰베이스 `아톰랩 → 위하고 → 수집 토큰 관리`에서 발급)
   - **수집 켜기** 토글 ON (기본 ON)

## 동작 확인

- 위하고에서 손익계산서 등을 조회하면, 팝업 **최근 수집**에 `손익계산서 ✅ 신규 저장` 으로 뜹니다.
- 같은 화면을 다시 열면 `변경 없음` 또는 전송 생략(내용이 같으면).
- 아톰베이스 `/atom-lab/wehago` 에서 해당 회사 스냅샷이 `source=extension` 으로 쌓입니다.

## 확장 ID (관리자용)

이 확장은 `manifest.json`에 고정 `key`가 들어 있어 **모든 PC에서 ID가 동일**합니다:

```
goecaigfmlcbomcdhdbglpfiejolhdbi
```

아톰베이스(Vercel) 환경변수에 아래를 등록하면 브라우저 CORS까지 닫힙니다(선택):

```
WEHAGO_EXTENSION_ORIGIN=chrome-extension://goecaigfmlcbomcdhdbglpfiejolhdbi
```

> MV3 확장 서비스워커의 fetch는 host_permissions로 CORS를 우회하므로 위 설정이 없어도 동작합니다.
> 토큰 인증이 1차 방어이고, origin 제한은 보강입니다.

## 수집 대상 화면

`sabc0102`(회사정보) · `sacl0106`(손익계산서) · `swsa0105`(급여대장) · `saas0106`(고정자산) · `swbu0111`(사업소득). 그 외 화면은 무시합니다.
